# دليل إعداد ال SMTP الخارجي (Production Ready)

إذا ظهرت لك رسالة **"email rate limit exceeded"**، فهذا يعني أنك تجاوزت الحد المسموح به (3 رسائل في الساعة) من قبل مزود البريد الافتراضي في Supabase. للإنتاج الحقيقي، يجب عليك استخدام مزود خارجي مثل **Resend**.

## الخطوة 1: إنشاء حساب في Resend
1. اذهب إلى [Resend.com](https://resend.com) وأنشئ حساباً.
2. قم بإضافة نطاقك (Domain) في قسم **Domains** وقم بتفعيل سجلات الـ DNS المطلوبة.
3. ابته إلى قسم **API Keys** وأنشئ مفتاحاً جديداً.

## الخطوة 2: الإعداد في Supabase
1. اذهب إلى لوحة تحكم [Supabase Dashboard](https://supabase.com/dashboard).
2. اختر مشروعك، ثم انتقل إلى **Project Settings** > **Auth**.
3. ابحث عن قسم **SMTP Settings**.
4. قم بتبديل مفتاح **Enable Custom SMTP** إلى وضع التفعيل (ON).

## الخطوة 3: إدخال بيانات الـ SMTP
أدخل البيانات التالية (الخاصة بـ Resend):

| الحقل | القيمة |
| :--- | :--- |
| **SMTP Host** | `smtp.resend.com` |
| **SMTP Port** | `465` (مع تفعيل SSL) أو `587` (بدون تفعيل SSL) |
| **SMTP User** | `resend` |
| **SMTP Password** | `مفتاح API الخاص بك (re_...)` |
| **Sender Email** | `البريد الرسمي لنطاقك` (مثال: `no-reply@yourdomain.dz`) |

## الخطوة 4: الحفظ والاختبار
1. اضغط على **Save**.
2. يمكنك الآن محاولة التسجيل في منصتك. ستلاحظ اختفاء مشكلة الـ Rate Limit تماماً، ووصول الرسائل بشكل فوري واحترافي.

---

> [!TIP]
> توفر Resend باقة **Free Tier** سخية جداً تسمح لك بإرسال 3,000 رسالة شهرياً مجاناً، وهو أكثر من كافٍ لمرحلة الـ MVP.

> [!IMPORTANT]
> تأكد من تحديث رابط توجيه المستخدم (Site URL) في إعدادات Supabase Auth ليكون رابط موقعك على Vercel وليس `localhost`.

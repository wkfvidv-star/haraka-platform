## 📝 **السيناريوهات المطلوبة**

### 🟢 **1. السيناريو الأساسي (Happy Path)**

#### **الخطوة 1.1: إنشاء جلسة تجريبية**
- **الهدف**: إنشاء session جديد بنجاح
- **المدخلات**:
  ```json
  {
    "studentName": "أحمد محمد التجريبي",
    "exerciseType": "كرة القدم",
    "uploadExpiresMinutes": 5
  }
  ```
- **النتيجة المتوقعة**: 
  - HTTP 201
  - session_id صالح
  - upload_url مع انتهاء صلاحية
  - audit_log مع is_demo=true

#### **الخطوة 1.2: رفع فيديو تجريبي**
- **الهدف**: رفع demo_sample.mp4 بنجاح
- **المدخلات**: ملف MP4 صالح (< 100MB)
- **النتيجة المتوقعة**:
  - HTTP 200
  - session.status = 'processing'
  - بدء Worker تلقائياً

#### **الخطوة 1.3: معالجة التحليل**
- **الهدف**: إكمال التحليل بالمحاكي
- **المدة المتوقعة**: 15-30 ثانية
- **النتيجة المتوقعة**:
  - session.status = 'completed'
  - report_id متاح
  - overall_score بين 75-95

#### **الخطوة 1.4: عرض التقرير**
- **الهدف**: عرض تقرير مفصل
- **النتيجة المتوقعة**:
  - metrics كاملة (balance, speed, accuracy, technique)
  - recommendations مع تمارين
  - insights ملونة
  - confidence_score > 0.8

#### **الخطوة 1.5: الإشعارات**
- **الهدف**: إرسال إشعارات لجميع الأدوار
- **النتيجة المتوقعة**:
  - 4 إشعارات (student, parent, teacher, coach)
  - جميع الإشعارات status = 'sent'

---

### 🔴 **2. سيناريوهات الفشل (Edge Cases)**

#### **الخطوة 2.1: ملف كبير الحجم**
- **الهدف**: رفض الملفات الكبيرة
- **المدخلات**: ملف > 100MB
- **النتيجة المتوقعة**:
  - HTTP 400
  - رسالة خطأ واضحة بالعربية
  - عدم إنشاء session

#### **الخطوة 2.2: تنسيق ملف غير مدعوم**
- **الهدف**: رفض الملفات غير MP4/MOV/AVI
- **المدخلات**: ملف PDF أو TXT
- **النتيجة المتوقعة**:
  - HTTP 400
  - رسالة "تنسيق غير مدعوم"
  - عدم المتابعة

#### **الخطوة 2.3: انتهاء صلاحية الرفع**
- **الهدف**: رفض الرفع بعد انتهاء الصلاحية
- **المدخلات**: محاولة رفع بعد 5 دقائق
- **النتيجة المتوقعة**:
  - HTTP 400
  - رسالة "انتهت صلاحية الرفع"

#### **الخطوة 2.4: فشل Worker (محاكاة)**
- **الهدف**: التعامل مع فشل المعالجة
- **المحاكاة**: إيقاف Worker مؤقتاً
- **النتيجة المتوقعة**:
  - session.status = 'failed'
  - error_message واضح
  - إشعار للمدير الفني

---

### 🔒 **3. اختبار الأمان والخصوصية**

#### **الخطوة 3.1: فصل بيانات Demo**
- **الهدف**: التأكد من عزل بيانات Demo
- **الفحص**:
  ```sql
  SELECT * FROM exercise_sessions_demo WHERE is_demo = false;
  SELECT * FROM analysis_reports_demo WHERE is_demo = false;
  SELECT * FROM audit_logs WHERE is_demo = false AND resource_type LIKE '%demo%';
  ```
- **النتيجة المتوقعة**: 0 سجلات

#### **الخطوة 3.2: عدم وجود PII حقيقية**
- **الهدف**: التأكد من عدم تسريب بيانات شخصية
- **الفحص**: مراجعة جميع التقارير المُولدة
- **النتيجة المتوقعة**: 
  - أسماء تجريبية فقط
  - لا توجد أرقام هواتف/إيميلات حقيقية
  - جميع البيانات محددة كـ demo

#### **الخطوة 3.3: صلاحيات الوصول**
- **الهدف**: التحقق من صلاحيات API
- **الفحص**: محاولة الوصول بدون صلاحيات
- **النتيجة المتوقعة**: HTTP 401/403 حسب الحالة

### ⚡ **4. قياسات الأداء**

#### **الخطوة 4.1: زمن إنشاء الجلسة**
- **القياس**: POST /api/demo/sessions
- **المتوقع**: < 2 ثانية
- **المقاييس**: P50, P95, P99

#### **الخطوة 4.2: زمن رفع الملف**
- **القياس**: POST /api/demo/sessions/{id}/complete
- **المتوقع**: < 5 ثواني لملف 10MB
- **المقاييس**: Upload latency

#### **الخطوة 4.3: زمن المعالجة (Mock)**
- **القياس**: من بداية المعالجة حتى إكمال التقرير
- **المتوقع**: 15-30 ثانية
- **المقاييس**: Processing time

#### **الخطوة 4.4: زمن استجابة التقرير**
- **القياس**: GET /api/demo/reports/{id}
- **المتوقع**: < 1 ثانية
- **المقاييس**: Response time

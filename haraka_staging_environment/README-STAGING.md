# بيئة التجريب لمنصة حركة 🚀

## 📋 نظرة عامة
هذه بيئة تجريبية منفصلة تماماً لاختبار الميزات الجديدة قبل دمجها في النسخة الرسمية.

## 🎯 الهدف
- اختبار التحديثات الجديدة بأمان
- تجربة ميزات الـ backend والـ database integration
- ضمان عدم التأثير على المستخدمين الفعليين
- إمكانية العودة السريعة للنسخة الرسمية

## 🔧 التشغيل السريع

### 1. تشغيل البيئة التجريبية
```bash
# تشغيل الواجهة الأمامية (المنفذ 5174)
npm run dev:staging

# تشغيل الـ API التجريبي (المنفذ 3001)
npm run api:staging

# تشغيل كامل (واجهة + API)
npm run staging:start
```

### 2. إعداد قاعدة البيانات التجريبية
```bash
# إعداد الجداول
npm run db:setup:staging

# إدراج بيانات تجريبية
npm run db:seed:staging
```

### 3. العودة للنسخة الرسمية
```bash
# العودة الفورية
npm run rollback:production

# أو من خلال الواجهة - الضغط على زر "العودة للنسخة الرسمية"
```

## 🔒 الأمان والعزل

### ✅ ضمانات الأمان:
- **قاعدة بيانات منفصلة**: `haraka_staging`
- **متغيرات بيئة منفصلة**: `.env.staging`
- **منافذ مختلفة**: 5174 للتجريب، 5173 للرسمي
- **مجلد منفصل**: `haraka_staging_environment`
- **شريط تنبيه**: يظهر في كل صفحة للتذكير

### 🚫 ما لا يتأثر:
- النسخة الرسمية تعمل بشكل طبيعي
- بيانات المستخدمين الحقيقيين محمية
- لا توجد تداخلات في الكود
